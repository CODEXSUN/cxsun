# Coding Rules

- Use TypeScript.
- Use explicit package public exports.
- Avoid circular dependencies.
- Avoid private cross-package imports.
- Keep framework business-free.
- Keep contracts implementation-free.
- Keep UI primitives workflow-free.
- Use typed errors.
- Use typed config.
- Use typed request and response contracts.
- Add docs with structural changes.
- Do not write files over 700 lines.
- Split files before they reach 700 lines.
- Keep one file focused on one responsibility.
- Do not overcode beyond the active task.
- Do not implement future-task behavior early.
