# Frontend Rules

- UI primitives live in `packages/ui`.
- Product or Platform workflows must not live in `packages/ui`.
- Feature folders own components, routes, API client usage, state, and docs.
- Standardize loading, empty, error, and success states.
- Forms must use a shared validation pattern.
- Route guards must use explicit auth/session contracts.
- Accessibility baseline is required for shared UI primitives.
